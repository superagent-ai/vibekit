## Manual Setup

The VibeKit MCP server can be used with any MCP-compatible client. The server will be automatically downloaded and run using npx - no installation required!

### MCP Configuration

Add this configuration to your MCP client:

```json
{
  "mcpServers": {
    "vibekit": {
      "command": "npx",
      "args": ["-y", "@vibe-kit/mcp-server"],
      "env": {}
    }
  }
}
```

<Note>
  The `-y` flag tells npx to automatically install the package if it's not already cached. If you have other MCP servers configured, add VibeKit alongside them in the `mcpServers` object.
</Note>


## Claude Desktop Setup

If you're using Claude Desktop specifically, here's how to set it up:

### Step 1: Locate Configuration File

Find your Claude Desktop configuration file:

<Tabs>
  <Tab title="macOS">
    ```bash
    ~/Library/Application Support/Claude/claude_desktop_config.json
    ```
  </Tab>
  <Tab title="Windows">
    ```bash
    %APPDATA%\Claude\claude_desktop_config.json
    ```
  </Tab>
  <Tab title="Linux">
    ```bash
    ~/.config/Claude/claude_desktop_config.json
    ```
  </Tab>
</Tabs>

### Step 2: Add Configuration

Add the MCP configuration shown above to your `claude_desktop_config.json` file.

### Step 3: Restart Claude Desktop

Restart Claude Desktop to load the new configuration.

Once configured, you can verify the MCP server is working by asking your AI assistant:


The assistant should respond with a list of your projects (or indicate no projects exist yet).

## Local Development

For local development of the MCP server:

### Clone and Build

```bash
# Clone the repository
git clone https://github.com/superagent-ai/vibekit
cd vibekit

# Install dependencies
npm install

# Build the MCP server
npm run build --workspace=packages/mcp-server
```

### Configure for Local Development

Update your MCP client config to use the local build:

```json
{
  "mcpServers": {
    "vibekit": {
      "command": "node",
      "args": ["/absolute/path/to/vibekit/packages/mcp-server/dist/index.js"],
      "env": {}
    }
  }
}
```

### Development Mode

Run in watch mode for development:

```bash
npm run dev --workspace=packages/mcp-server
```


## Alternative Transports

### Stdio (Default)

The default configuration uses stdio transport, which works with most desktop MCP clients:

```json
{
  "mcpServers": {
    "vibekit": {
      "command": "npx",
      "args": ["-y", "@vibe-kit/mcp-server"]
    }
  }
}
```

### HTTP Stream

For web-based clients or remote access, use HTTP stream transport:

```json
{
  "mcpServers": {
    "vibekit": {
      "command": "npx",
      "args": [
        "-y",
        "@vibe-kit/mcp-server",
        "--transport", "http-stream",
        "--port", "8080"
      ]
    }
  }
}
```

Then connect your client to: `http://localhost:8080/mcp`

## Troubleshooting

### Server Not Found

If your MCP client can't find the VibeKit server:

1. Check the configuration file path is correct
2. Ensure the JSON syntax is valid
3. Restart your MCP client completely
4. Check the client's logs for error messages

### Permission Errors

If you encounter permission errors:

```bash
# Ensure the projects directory exists and is writable
mkdir -p ~/.vibekit
chmod 755 ~/.vibekit
```

### Debug Mode

Enable debug logging by setting the environment variable:

```json
{
  "mcpServers": {
    "vibekit": {
      "command": "npx",
      "args": ["-y", "@vibe-kit/mcp-server"],
      "env": {
        "DEBUG": "vibekit:*"
      }
    }
  }
}
```
